**A note-taking system that requires discipline to maintain will always fail.** The PARA method works because it organizes by *actionability*, not topic — so filing a note takes two seconds and finding it later takes five. This recipe builds the full structure around your specific life.

## The Recipe

```
Act as an expert systems architect specializing in building Personal Knowledge Management (PKM) ecosystems using Tiago Forte's PARA Method (Projects, Areas, Resources, Archives). I want to migrate my chaotic digital notes, ideas, and files into a clean, sustainable local Obsidian vault structure that eliminates filing friction and maximizes retrieval speed.

My primary domains of life, active projects, and hobbies are: [INSERT DETAILS, e.g., Cloud Data Engineering portfolio, 9 perennial garden beds, tasting journals, retro vinyl collection].

Please build a comprehensive "PKM Vault Blueprint" tailored entirely to my life:
1. The Folder Hierarchy Map: Provide a visual folder and sub-folder structure showing exactly how to organize my inputs across the 4 PARA categories.
2. The Metadata Template (Frontmatter): Generate a clean YAML markdown template to paste at the top of my project and resource notes, using tags and link structures optimized for easy querying.
3. The "Weekly Ingest" Protocol: Give me a lean, 5-minute routine to process a chaotic, raw inbox folder and move notes cleanly into the vault without falling into over-organization paralysis.
```

## The four PARA categories

| Category | What goes here | Examples |
|---|---|---|
| **Projects** | Active work with a defined end state | Portfolio piece, garden redesign, trip planning |
| **Areas** | Ongoing responsibilities with no end date | Health, finances, career, relationships |
| **Resources** | Topics of interest for future reference | Vinyl collecting guides, plant databases, recipes |
| **Archives** | Completed or inactive items from the other three | Finished projects, past job notes, old resources |

## Why PARA works when topic-based systems fail

Topic folders demand a decision every time you file something: *is this "design" or "branding" or "visual assets"?* PARA demands only one decision: *is this active and has an end date (Project), ongoing (Area), reference (Resource), or done (Archive)?* The simpler the filing decision, the more likely you are to actually file things.

## The Weekly Ingest Protocol — the critical habit

A vault with no regular maintenance becomes another pile. The 5-minute ingest routine processes your inbox without the perfectionism trap:
1. Open the inbox folder
2. For each note: assign it to one of the four PARA buckets, add frontmatter, move it
3. If you can't decide in 5 seconds, it goes to Resources
4. Close the inbox — don't reorganize what's already filed

The rule "if you can't decide in 5 seconds, it goes to Resources" is the anti-perfectionism constraint that keeps the habit sustainable.
