**Role: The Personal Infrastructure Designer.** Objective: close the gap between a strategy on paper and daily behavior — by embedding the quarterly plan directly into your calendar, weekly review, and note-taking system.

## The Recipe

```
Act as a personal infrastructure designer and behavioral habits architect. I have a flawless quarterly strategy on paper, but I need to integrate it directly into my real-world daily and weekly operations so it doesn't get forgotten in a digital archive.

My 12-week sprint plan is: [PASTE ROADMAP OR OVERVIEW HERE]
My primary productivity stack includes: [e.g., local Obsidian folders using the PARA method, a simple note app, standard calendar blocking]

Please design an operational "Daily & Weekly Alignment Framework" featuring:
- The Weekly Review Audit Trigger: Provide a 4-step checklist to run every Sunday to audit my current sprint progress and set my upcoming weekly "Rule of 3" high-leverage outcomes.
- Time-Block Architecture: Show me exactly how to block out my calendar weekly (e.g., scheduling two 90-minute deep work blocks before checking any communication channels).
- The Local Vault Sync: Give me a clean markdown template to paste into my daily note system or Obsidian environment to track leading habit indicators natively without adding logging friction.
```

## The three integration components

| Component | What it creates |
|---|---|
| Weekly Review Audit Trigger | A repeatable 4-step Sunday ritual that closes the week and sets the next one |
| Time-Block Architecture | A calendar structure that protects deep work from being colonized by communications |
| Local Vault Sync | A daily note template that tracks leading indicators without adding friction |

## The Weekly Review — the ritual that keeps everything else alive

Without a weekly review, quarterly plans live in a document and die there. The Sunday audit forces a brief but honest reckoning: Did I hit this week's milestones? What's the sprint milestone for this coming week? What's my Rule of 3 for Monday, Tuesday, Wednesday? Four steps, 15 minutes, and the quarter stays on track.

## The Rule of 3 weekly framing

Instead of a task list, the weekly alignment produces three high-leverage outcomes — the three things that, if completed, make the week a success regardless of everything else that happened. This framing prevents the cognitive overload of a 30-item list and creates a clear, achievable success condition for every week.

## Time-Block Architecture — the non-negotiable structure

The architecture typically produces a weekly calendar template with:
- **Deep work blocks** (90 minutes, 2x daily minimum) scheduled before communications are opened
- **Admin/communications window** batched to specific afternoon slots
- **Weekly review** blocked on Sunday evening or Monday morning
- **Sprint checkpoint** blocked at the end of each 3-week period

The specific structure is generated from your productivity stack — Obsidian + PARA produces different recommendations than a standard calendar + to-do app.

## The Local Vault Sync — no extra friction

The markdown template is designed to be pasted into a daily note, not a separate tracking system. It logs the leading indicators from Phase 3's OKR Matrix — "Deep work blocks completed today: __ / 2" — in the same file where daily notes already live. Tracking that requires opening a separate app has a 90% failure rate within two weeks. Tracking that lives in the daily note you already write has a much higher survival rate.

## Closing the loop: Phase 5 → Phase 1

The daily notes and weekly review produce the raw data that feeds Phase 1 next quarter. The execution rate, energy drain, systemic friction, and peak-performance conditions you'll be asked about in the next post-mortem are all observable in the vault if you've been running the daily sync. The system is designed as a loop, not a linear sequence.
