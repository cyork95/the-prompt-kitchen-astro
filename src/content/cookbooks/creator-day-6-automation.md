**The content is done. Now make sure you never manually schedule, cross-post, or dig through folders to find it again.** Day 6 builds the system that runs while you create.

## The Recipe

```
Act as an elite systems engineer and automation architect for content businesses. I want to build an automated, zero-friction system to handle the scheduling, cross-posting, and long-term repurposing of this week's content assets so I don't get trapped in administrative work.

My current tool stack is: [INSERT STACK, e.g., Notion, Buffer, Zapier, Make, Proton Drive, Obsidian]

Please build a complete systems architecture for my workflow, detailing:
- The Central Database Structure: How to tag and organize this week's assets (Pillar, Atomized, Platform, Status, Visual Assets) in a clean directory or repository for future reuse.
- The Automation Loops: Provide a step-by-step logic model for 2 automated workflows (e.g., triggering cross-posts when a primary asset goes live, or auto-moving assets from "Draft" to "Scheduled").
- The Evergreen Vault Protocol: Explain a systematic method for automatically identifying if this week's content performs well, and how to queue it to re-air as an "evergreen" asset 3 to 6 months from now.

Keep the setup lean, practical, and highly scalable.
```

## The three systems this builds

| System | What it does |
|---|---|
| Central Database | Tags every asset so you can find and reuse it later |
| Automation Loops | Handles scheduling and cross-posting without manual steps |
| Evergreen Vault | Resurfaces high-performing content at 3–6 month intervals automatically |

## Don't over-engineer Day 6

Ask for exactly what your current stack supports. A two-tool setup (Notion + Buffer) needs a different architecture than a five-tool setup. The prompt adapts — just be honest about what you actually use.
